### AWS Lightsail 인스턴스에서 CPU 크레딧 소진 3분 전 커널 패닉 발생 이유

#### 세부 질문
- 특정 리전 DNS 장애 vs 일반적인 캐시 무효화 지연 원인에 대한 관점 차이.
- AWS Lightsail 인스턴스의 CPU 크레딧 사용과 관련된 최적 조건.

#### 실패 원인 및 비교 기준
- **DNS 장애**: 새벽 3시에 AWS 리전 DNS가 정상적으로 작동하지 못해, AWS Lightsail 인스턴스에서 CPU 크레딧 소진이 발생했습니다.
- **캐시 무효화 지연**: 일반적인 경우는 AWS Lightsail 인스턴스가 클라우드 서비스를 사용하는 동안 데이터 캐시가 일정 시간 동안 효력이 없어졌습니다.

#### 조건 및 절차
1. **DNS 장애 발생**: 새벽 3시에 AWS 리전 DNS가 정상 작동하지 않는 상황에서, 특정 AMI 조합을 사용한 AWS Lightsail 인스턴스는 CPU 크레딧 소진이 발생했습니다.
2. **캐시 무효화 지연**: 일반적인 경우는 AWS Lightsail 인스턴스가 클라우드 서비스를 사용하는 동안 데이터 캐시가 일정 시간 동안 효력이 없어졌습니다.

#### 예외 및 관찰 포인트
- **DNS 장애 예외**: 특정 리전 DNS가 정상 작동하지 않은 상황에서 AWS Lightsail 인스턴스는 CPU 크레딧 소진을 겪었으며, 해당 AMI 조합 사용이 원인일 수 있습니다.
- **캐시 무효화 지연 관찰 포인트**: 일반적으로 AWS Lightsail 인스턴스의 CPU 크레딧 사용과 관련된 최적 조건을 확인해 보았습니다. 그러나 이 경우는 DNS 장애에 따른 결과였음을 볼 수 있었습니다.
